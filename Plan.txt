# Practice Project: Enterprise Change Request Manager

## 1. Project Goal

Build a Python application that reads enterprise change requests from a JSON file, analyzes them, and generates reports for a Change Advisory Board.

This project should prove that you can independently use:

* Python classes
* JSON file handling
* Clean multi-file project structure
* Error handling
* Git workflow
* README documentation

No AI API should be used in this project.

---

# 2. Business Scenario

You are part of an Enterprise Support team.

Customers submit change requests for planned activities such as:

* Storage firmware upgrades
* Volume migrations
* CIFS/NFS configuration changes
* Security patching
* Maintenance windows
* Disaster recovery testing
* Backup policy changes

Your job is to build a system that helps the Change Advisory Board quickly understand:

* How many change requests exist
* Which changes are high risk
* Which changes are scheduled
* Which changes are completed
* What customer notifications should be sent


## Required Methods

### Method 1: `is_high_risk()`

Purpose:

Check whether the change request is high risk.

Expected behavior:

```text
If risk is "High" → return True
Otherwise → return False
```

Example:

```python
change.risk = "High"
change.is_high_risk()
```

Expected result:

```python
True
```

---

### Method 2: `generate_summary()`

Purpose:

Create a short internal summary for support/CAB teams.

Expected format:

```text
CR1001 - ABC Bank - High Risk - Scheduled - Upgrade ONTAP firmware on production cluster
```

Format rule:

```text
change_id - customer - risk Risk - status - description
```

---

### Method 3: `to_dict()`

Purpose:

Convert the object back into a dictionary so it can be saved into JSON output.

Expected output example:

```json
{
    "change_id": "CR1001",
    "customer": "ABC Bank",
    "risk": "High",
    "status": "Scheduled",
    "description": "Upgrade ONTAP firmware on production cluster",
    "planned_days": 3
}
```

This method is important for saving high-risk changes into JSON.

---

### Method 4: `generate_customer_notification()`

Purpose:

Generate a customer-friendly update message.

Expected format:

```text
Dear ABC Bank,

Your change request CR1001 for "Upgrade ONTAP firmware on production cluster" is currently Scheduled.

Planned implementation window: within 3 days.
Risk Level: High

We will keep you updated throughout the change lifecycle.

Regards,
Enterprise Support Team
```

Rules:

* Use the customer name.
* Use the change ID.
* Use the description.
* Use the current status.
* Use planned_days.
* Use risk.
* Message should be professional.
* Do not overpromise.
* Do not say the change is completed unless status is actually Completed.

# 8. file_handler.py Requirements

Create a class:

```python
class Files:
```

The class should manage JSON loading and writing.

## Method 1: `load_from_json()`

Purpose:

Read JSON data from the input file.

Expected behavior:

* If file exists and JSON is valid, return data.
* If file is missing, print an error and return empty list.
* If JSON is invalid, print an error and return empty list.
* For any other error, print a safe error message and return empty list.

Handle:

```python
FileNotFoundError
json.JSONDecodeError
Exception
```

---

## Method 2: `dump_to_json(data, output_file)`

Purpose:

Write data to a JSON file.

Rules:

* Use `indent=4`.
* Should accept output file path.
* Should write lists and dictionaries.
* Should print a success message or return nothing.

```text
8. Save all three outputs
9. Print final success message
```
